# 象棋学习 MVP 骨架（本地无依赖版）

## 已实现
- 注册/登录
- 多用户会话切换（本地存储）
- 家庭组创建与邀请码加入
- 成员角色（`parent`/`child`）
- 家庭组约束：
  - 每个用户只能创建一个家庭组
  - 孩子账号只能加入一个家庭组
- 权限规则：
  - 仅可看本人数据
  - `parent` 可看同家庭 `child` 数据
  - `child` 不可看家长数据
- 家庭组维护（仅创建者）：
  - 改名
  - 重置邀请码
  - 解散家庭组（保留个人学习数据，只删除群组与成员关系）
- 人机对战 / 棋局录入与回放：
  - 当前用户可新建并保存多个对局
  - 支持模式切换：`practice`（手动双边录入）/ `ai`（人机对战）
  - 人机模式可选择电脑执子方与难度（入门/标准/进阶）
  - 棋盘点击录入走子（含合法性校验）
  - 上一步/下一步/首步/末步回放
  - 按象棋读法生成基础棋谱记录（如 `炮五平七` 结构）
  - 记谱 V2：同一路同类子时支持“前/后/中”消歧（如“前马退六”）
- 每步自动评估：是否接近最优、与最优差值、下一步风险（被吃/被将）
  - 评估 V2：采用统一评分口径，风险提示升级为“落点净亏风险”（避免把正常兑子误判为风险）

- 复盘基础：
  - 一键生成复盘摘要（高频问题、建议、逐手标签）
  - 自动标签（得子、战术得子、反复调子、应将、送子风险）
  - 手动标签与备注（按步号打标签）
- 对战（双账号轮换）：
  - 创建对战房间并生成邀请码
  - 另一账号可凭邀请码加入（红黑双方轮流走子）
  - 对战双方各自仅可操作本方棋子
  - 支持对战回放（首步/上一步/下一步/末步）
  - 对战棋谱按象棋读法记录与展示
  - 每步自动评估：最优性 + 下一步风险

## 走子合法性（当前已支持）
- 各子基本走法：车、马、相/象、仕/士、帅/将、炮、兵/卒
- 马腿、象眼、炮架等关键限制
- 九宫限制（帅/将、仕/士）
- 兵卒过河前后走法差异
- 将军校验：禁止“送将”
- 将帅照面约束（含飞将吃将场景）

## 引擎说明
- 当前内置：浏览器端轻量 Alpha-Beta 搜索（低开销，适合 MVP）
- 已引入 GitHub 开源算法：`xqbase/xqwlight`（JavaScript 引擎）用于“最优着法建议”和人机走子增强
- 难度档位通过搜索深度与随机性控制，且与“每步最优评估”共用一套计算能力
- 可对接的 GitHub 强引擎（后续）：
  - `official-pikafish/Pikafish`（UCI 象棋强引擎）
  - `xqbase/xqwlight`（已接入当前前端版）

## 第三方依赖
- `mvp/vendor/xqwlight/*` 来自 `xqbase/xqwlight`（GPL-2.0），保留原许可证文件。

## 运行方式
- 推荐：在仓库根目录启动后端化服务 `npm start`，然后访问 `http://localhost:3001`
- 兼容：也可直接打开 `mvp/index.html`（此时仅本机缓存，不走共享后端）

## 后端化说明（V1）
- 新增零依赖 Node 服务：`server/index.js`
- 共享数据存储：`server/data/db.json`（运行时自动创建）
- 前端数据层：`mvp/js/store.js` 已切换为“远端优先、本地回退”
- 登录会话仍保留在本地浏览器，避免家庭成员互相挤掉登录状态
- 首次连接到空后端时，会自动把旧本地数据迁移上去（一次性）

## 回归测试
- 命令行（Node 16+）：在仓库根目录运行 `npm run test:regression`
- 浏览器：打开 `mvp/tests/assessment-regression.html`
- 覆盖场景：开局常见着法、红黑记谱格式、前后子消歧、吃将判定、外部最优不一致降级、净亏风险识别与等价兑子过滤

## 说明
- 这是 MVP 权限骨架，当前用 `localStorage` 模拟数据库。
- 后续迁移到正式后端时，接口和规则保持不变，仅替换数据层。
