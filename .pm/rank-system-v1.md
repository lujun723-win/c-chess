# 段位评分规则 V1（六维雷达）

## 设计目标
- 用“可解释、可追踪”的方式评估棋力成长，不只看单一胜率。
- 兼顾你提出的关键维度：积分累计、胜负率、电脑难度、子力强度。
- 输出两类结果：
  1. 段位（入门/业余段位）
  2. 六维雷达（能力画像）

## 参考依据（外部）
- Elo/FIDE：用“实际得分 vs 期望得分”修正评分，避免只看胜率。  
  参考：FIDE Rating Regulations（8.1/8.3）  
  https://handbook.fide.com/chapter/B022024
- Glicko-2：引入不确定性（RD）和波动（volatility），降低样本少时的误判。  
  参考：Mark Glickman《Example of the Glicko-2 system》  
  https://glicko.net/glicko/glicko2.pdf
- TrueSkill：保守评分思想（考虑不确定性后的保守估计）。  
  参考：Microsoft TrueSkill  
  https://www.microsoft.com/en-us/research/project/trueskill-ranking-system/

> 说明：下面“六维定义和权重”是针对中国象棋学习场景的产品化推导，不是直接照搬任何单一官方标准。

## V1 六维（0-100）

1. 攻击效率（Attack）
- 含义：把进攻机会转化为实质优势的能力。
- 子指标：
  - 进攻战术题正确率（40%）
  - 优势局转胜率（35%）
  - 主动将军后收益率（25%）

2. 计算精度（Calculation）
- 含义：每步决策与引擎最优解的接近程度。
- 子指标：
  - 最优步匹配率（60%）
  - 平均引擎损失分 EPL（40%，损失越低越好）

3. 子力运用（Piece Efficiency）
- 含义：子力活跃、交换质量、残局转化能力。
- 子指标：
  - 子力活跃度（35%）
  - 交换收益效率（35%）
  - 残局转化率（30%）

4. 防守韧性（Defense）
- 含义：抗压、止损、避免崩盘的能力。
- 子指标：
  - 致命失误率（越低越好，40%）
  - 劣势局止损率（35%）
  - 防守题正确率（25%）

5. 对抗强度（Difficulty Adaptation）
- 含义：面对强对手/高难度电脑时的表现质量。
- 子指标：
  - 实际得分-期望得分表现差（50%）
  - 平均对手强度分（25%）
  - 高难度电脑对局得分（25%）

6. 成长执行（Growth & Consistency）
- 含义：长期进步能力（你提到的“积分累计”核心放在此维度）。
- 子指标：
  - 成长积分累计（50%）
  - 训练连续性（30%）
  - 复盘完成率（20%）

## 成长积分（GP）规则（V1）

每局积分增量（按对局）：

`GP_delta = max(0, EloDelta) * M_difficulty + ReviewBonus + TrainingLinkBonus`

- `EloDelta`：基于实际得分与期望得分的变化量（可正可负，负值不计入 GP）
- `M_difficulty`：
  - 人类对手：`1 + 0.15 * OppStrengthNorm`
  - 电脑对手：`1 + 0.20 * AIDifficultyNorm`
- `ReviewBonus`：完成该局复盘 +2
- `TrainingLinkBonus`：完成该局关联训练 +1

说明：  
- 胜率被纳入 EloDelta（实际 vs 期望），不是孤立计算。  
- 打弱对手的“水胜率”收益会明显低于打强对手或高难度电脑。

## 评分总分与段位

### 六维总分
`TotalScore = 0.16*A + 0.20*C + 0.18*P + 0.18*D + 0.16*O + 0.12*G`

其中：  
- `A` 攻击效率  
- `C` 计算精度  
- `P` 子力运用  
- `D` 防守韧性  
- `O` 对抗强度  
- `G` 成长执行

### 稳定性修正（防抖）
`FinalScore = TotalScore - UncertaintyPenalty`

- `UncertaintyPenalty = clamp(RD / 20, 0, 8)`
- RD 越高，说明样本不足或表现波动大，先保守给分。

### 段位映射（V1）
- 0-29：入门准备
- 30-39：入门
- 40-49：业余 1 段
- 50-59：业余 2 段
- 60-69：业余 3 段
- 70-79：业余 4 段
- 80-89：业余 5 段
- 90-100：业余 6 段

### 升降段规则（V1）
- 升段条件：`FinalScore` 连续 20 局高于目标段阈值，且 `RD < 90`。
- 降段条件：`FinalScore` 连续 30 局低于当前段阈值 - 4 分。
- 目的：减少短期波动导致的频繁跳段。

## 雷达图展示规范
- 六轴固定：`攻击 / 计算 / 子力 / 防守 / 对抗 / 成长`
- 范围：0-100
- 显示：
  - 当前值
  - 30 天均值（虚线）
  - 段位目标线（可选）

## V1 所需数据（MVP）
- 对局结果、对手类型与强度、电脑难度
- 每步评估（或每局关键步评估）用于 EPL/最优步匹配
- 复盘记录、训练记录、连续性记录
- 子力相关统计（活跃度/交换/残局）

## V1 风险与后续
- 风险：EPL 与子力统计依赖引擎与特征提取质量。
- V2 方向：引入开局/中局/残局分段评分，加入时间管理维度（读秒稳定度）。
